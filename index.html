What is PincerCraft?

A fork of Mindcraft that keeps everything good and adds order on top.

QUE

Prioritized action queue

One serial lane handles every signal — chat, game events, self-prompts. Four queue modes decide what happens when input arrives mid-task.

COC

Live Code of Conduct + memory

Rules live in a writable book on a lectern — edit it in vanilla Minecraft and new rules apply in ~2 seconds. Layered memory (server · bot · per-player) persists what it learns across reboots.

PLN

Plan Mode, like Claude Code

Before acting, the bot uses read-only tools to look around and drafts a plan with real coordinates. No more (0,0,0) hallucinations.

EXE

Discovery-first executor

It perceives before it targets — search, scan, check inventory — then bakes real coordinates into one action. Long jobs route to role-based subagents: miner, builder, scout.

CYC

Self-improving loop

A task-giver invents a challenge, the bot attempts it, metrics score the run, an analyzer diagnoses, and an improver patches the code — on a branch, for a human to approve. Nothing auto-merges.

EVL

Spec-proof evaluation

Success is graded deterministically from a world-state snapshot — never the bot's own word, which it's been caught faking. Delta checks and a train/eval split keep every gain honest.

How does it work?

Every signal goes through one ordered lane. The LLM only ever sees one task at a time.

01 · INPUT

Signal arrives

Chat, game event, or self-prompt — all enter the same lane.

02 · ROUTE

Queue mode picked

Orchestrator picks interrupt, steer, followup, or collect.

03 · PLAN

Look before leaping

For real tasks, the bot enters Plan Mode and reads the world before writing any action.

04 · EXECUTE

Act, with COC on top

Tools run via AbortController. The Code of Conduct sits at the top of every prompt.

The four queue modes

ModeBehaviorWhen to use
interrupt Abort the current run, switch to the new input immediately. "Stop, come here."
steer Inject the new input into the current run between tool calls. "Also pick up dirt while you're at it."
followup Let the current run finish, then start the new input. "After the build, go to bed."
collect Merge multiple follow-ups into one batched turn. A burst of instructions handled together.
